## 第一部分：开发规则

### 一、节点开发规则

#### 1.1 文件结构
```
src/components/nodes/节点名/
├── 节点名.tsx        ← UI渲染
├── 节点名.logic.ts   ← 业务逻辑
├── 节点名.config.ts  ← 配置常量
└── index.ts         ← 统一导出
```

#### 1.2 必须用的钩子
| 场景 | 钩子 |
|:-----|:-----|
| 选厂家+模型 | useModelSelector(capability) |
| 选工作流模板 | useWorkflowSelector(capability, source) |
| 参数映射 | useParamMapper(params, 'api'\|'workflow') |
| 读上游数据 | getUpstreamData(nodeId, edges, nodes) |

#### 1.3 输入输出标准
```javascript
// 输入
data.inputs = { prompt?, fileUrl?, refImages?, ... }

// 输出（所有节点统一格式）
data.outputs = {
  type: 'image' | 'video' | 'audio' | 'text',
  output: URL | string,
  image?: URL?,
  video?: URL?,
  audio?: URL?,
  text?: string?
}

// 读上游（统一）
const { text, image, video, audio, all } = getUpstreamData(id, edges, nodes);
```

#### 1.4 样式规范
| 项 | 值 |
|:---|:----|
| 尺寸 | 240x240 |
| 圆角 | 16px |
| 边框 | 1px solid rgba(255,255,255,0.08) |
| 选中边框 | 1.5px solid rgba(168, 85, 247, 0.85) |
| Handle | 24px圆形，紫色边框，居中 |

#### 1.5 存储规范
所有文件走 MinIO：http://localhost:19000/workflows/xxx  
不用 db://，不用 IndexedDB。

#### 1.6 文本框规范
| 规则 | 说明 |
|:-----|:------|
| 最小高度 | 至少显示4行后才出滚动条 |
| 右键菜单 | 不拦截，支持原生复制/粘贴 |
| @提及 | 统一支持 |
| 样式 | 深色背景、圆角8px、白色字体 |

### 二、代码修改规则

#### 2.1 修改前
```bash
git add . && git commit -m "开工前备份"
```

#### 2.2 新功能用分支
```bash
git checkout -b feature/功能名
```

#### 2.3 命名规则
- 变量/函数：camelCase
- 组件：PascalCase
- localStorage key：`canvas_` 前缀
- 自定义事件：`canvas-` 前缀 + 中划线
- 文件名：功能清晰

### 七、统一执行入口（executeAction）
所有工具按钮、工具栏、lightbox、宫格、框选走同一个入口：
```javascript
async function executeAction({
  mediaType: 'text'|'image'|'video'|'audio',
  actionType: 'generate'|'upscale'|'grid'|'compare'|'saveAsNode',
  source: 'api'|'runninghub'|'local_comfyui',
  mediaUrl?: string,
  params?: Record<string, any>,
  callback?: (result) => void
})
```

### 八、后端API返回格式统一
```
成功 → { success: true, data: { ... } }
失败 → { success: false, error: "错误信息" }
```

- Events 统一用 `canvas-` 前缀 + 中划线
