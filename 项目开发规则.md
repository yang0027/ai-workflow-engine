# 项目开发规则

> 所有开发和修改必须遵守以下规则

---

## 一、节点开发规则

### 1.1 文件结构
每个节点拆成3个文件：
```
src/components/nodes/节点名/
├── 节点名.tsx        ← UI渲染
├── 节点名.logic.ts   ← 业务逻辑
├── 节点名.config.ts  ← 配置常量
└── index.ts         ← 统一导出
```

### 1.2 必须用的钩子
| 场景 | 钩子 |
|:-----|:-----|
| 选厂家+模型 | useModelSelector(capability) |
| 选工作流模板 | useWorkflowSelector(capability, source) |
| 参数映射 | useParamMapper(params, 'api'\|'workflow') |
| 读上游数据 | getUpstreamData(nodeId, edges, nodes) |

### 1.3 输入输出
```
输入: data.inputs = { prompt?, fileUrl?, refImages?, ... }
输出: data.outputs = { type('image'|'video'|'audio'|'text'), output, image?, video?, audio?, text? }
```

### 2. 强注释规范

任何模块、节点、关键数据转换函数（如 `getConnectedVal`）、适配器逻辑，必须添加详细的**中文注释**，重点写明：

- **输入输出格式**（例如：输入是 `minio` 还是 `base64` / `URL`，输出是何种结构）。
- **功能分块标识**（使用明显的分界注释，如 `// === 功能分块：IndexedDB 交互 ===`）。
- minio 端口 ：19000

### 1.4 样式

- 尺寸：240x240
- 圆角：16px
- 边框：1px solid rgba(255,255,255,0.08)
- 选中边框：1.5px solid rgba(168, 85, 247, 0.85)
- Handle：24px圆形

### 1.5 存储
所有文件走 MinIO：http://localhost:19000/workflows/xxx
不用 db://，不用 IndexedDB。

## 二、代码修改规则

### 2.1 修改前
```bash
git add . && git commit -m "开工前备份"
```

### 2.2 新功能用分支
```bash
git checkout -b feature/功能名
# 开发完成后
git checkout main
git merge feature/功能名
```

---

## 三、数据流规则

### 3.1 统一读上游
```javascript
const { text, image, video, audio, all } = getUpstreamData(id, edges, nodes);
```
禁止各节点自己写读取逻辑。

### 3.2 统一参数映射
```javascript
const params = useParamMapper(data.inputs, 'api');
// 所有节点按统一规则表映射
```

### 3.3 统一输出
```javascript
data.outputs = { type: 'image', output: 'http://...', image: 'http://...' }
```
禁止各节点自创输出格式。

---

## 四、命名规则

- 变量/函数：camelCase（驼峰）
- 组件/类：PascalCase（大驼峰）
- localStorage key：`canvas_` 前缀
- 自定义事件：`canvas-` 前缀 + 中划线
- 文件名：功能清晰，避免模糊命名
