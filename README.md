# AI Workflow Orchestration Engine (AI 工作流编排引擎)

> **核心定位：** 可视化节点拖拽画布 + Agent 协同调度 + Skill 插件体系的 AI 应用母工具（三明治架构体系）。

---

## ── 项目架构全景 ──

本项目采用**高内聚、低耦合**的“三明治”模块化架构，彻底与传统单体架构解耦。各个子系统之间仅通过结构化 API/事件进行通信，支持 Git 独立版本管理与无损回退。

```
                    ┌────────────────────────────────────────────────────────┐
                    │                   前端画布层 (Canvas)                    │
                    │   ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐       │
                    │   │ LLM  │ │ 生图  │ │ 视频  │ │ 条件  │ │ 循环  │ ...  │
                    │   │ 节点  │ │ 节点  │ │ 节点  │ │ 节点  │ │ 节点  │       │
                    │   └──────┘ └──────┘ └──────┘ └──────┘ └──────┘       │
                    │               ↓ 拖拽/连线/编排 (React Flow)             │
                    ├────────────────────────────────────────────────────────┤
                    │                   API 核心网关层                        │
                    │   接收画布请求 → 解析依赖拓扑结构 → 分发到对应能力模块   │
                    ├────────────────────────────────────────────────────────┤
                    │                  能力服务层 (Engine)                     │
                    │   ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐            │
                    │   │剧本  │ │角色  │ │分镜  │ │视频  │ │配音  │ ...      │
                    │   │服务  │ │服务  │ │服务  │ │服务  │ │服务  │            │
                    │   └─────┘ └─────┘ └─────┘ └─────┘ └─────┘            │
                    ├────────────────────────────────────────────────────────┤
                    │                   Skill 插件驱动层                      │
                    │   依据 Open Design (SKILL.md) 规范定义的专家级工作流    │
                    └────────────────────────────────────────────────────────┘
```

## ── 目录结构布局 ──

```text
ai-workflow-engine/
├── canvas/                    # 【前端画布】独立 SPA (React + TS + Vite + React Flow)
│   ├── src/
│   │   ├── components/       # 自定义组件 (微光磨砂玻璃风 UI)
│   │   ├── nodes/            # 预设画布节点 (LLM, Image, Video, TTS)
│   │   └── App.tsx
│   └── package.json
├── gateway/                   # 【API 网关】(Fastify + TypeScript + WS)
│   ├── src/
│   │   ├── routes/           # RESTful 接口路由
│   │   └── app.ts
│   └── package.json
├── engine/                    # 【能力服务】(Fastify 独立服务层)
│   ├── services/
│   │   ├── script/           # 剧本生成能力 (LLM 驱动)
│   │   ├── character/        # 角色设计与固定三视图
│   │   ├── storyboard/       # 智能分镜解析 (含宫格切割)
│   │   ├── video/            # 视频驱动接口 (ToonFlow / ComfyUI / RunningHub)
│   │   └── tts/              # 音频配音 (Fish.audio 接口，声音一致性绑定)
│   └── package.json
├── skills/                    # 【Skill 插件库】(实时热更新隐藏库)
│   ├── short-drama/          # 短剧生成专家 Skill (含 SKILL.md & workflow.json)
│   └── teaching/             # 教学视频制作 Skill
├── docs/                      # 【架构与设计白皮书】
│   └── architecture-arc42.md # 基于 arc42 架构模板的系统白皮书
└── README.md                  # 本说明文件
```

## ── 快速启动与激活 ──

### 步骤 1：打开工作空间
在您的 Antigravity 编辑器（或 VS Code 宿主端）中，选择 **“打开文件夹 (Open Folder)”**，定位并打开本目录：
`E:\开发\ai-workflow-engine`

### 步骤 2：启动前台开发套件
在聊天输入框内键入 `/help` 以唤醒 **Superpowers** 插件，或直接通过对话框命令我开启 `/superpowers:brainstorm` 针对 **Phase 1 (核心画布)** 的数据协议设计。
